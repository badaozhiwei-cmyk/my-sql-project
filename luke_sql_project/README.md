# 🚀 Luke Barousse SQL 实战项目：架构与实战手册

本项目通过对 2023 年全球 78 万条数据分析职位的深度挖掘，旨在掌握 PostgreSQL 实战技巧。

## 🧠 核心架构图解 (基于软件操作的比喻)

为了理解本项目是如何运行的，我们可以将整套系统拆解为以下四个层次：

### 1. 运行环境层 (Docker)
> **比喻：虚拟运行环境 / 绿色版软件**
*   **作用**：它像一个独立的“沙盒”，为数据库提供运行所需的操作系统环境。
*   **对应**：你运行的 `post_study` 容器。

### 2. 服务引擎层 (PostgreSQL)
> **比喻：后台服务 / 软件引擎**
*   **作用**：它是整个系统的“大脑”，负责处理数据存取逻辑，没有图形界面，静默守候在 **5432 端口**。
*   **对应**：Docker 镜像里跑着的 `postgres:latest`。

### 3. 持久存储层 (Volumes)
> **比喻：存档文件 / 硬盘分区**
*   **作用**：软件运行在内存，但数据必须存在硬盘。Volume 确保了即使 Docker 删了，数据依然在你的硬盘里。
*   **对应**：Docker 挂载的数据卷。

### 4. 交互界面层 (SQLTools)
> **比喻：客户端 / 浏览器外壳**
*   **作用**：通过“端口+密码”拨号连接到后台服务，将枯燥的数据库代码转化成我们可以点击、查看的表格。
*   **对应**：VS Code 左侧的 SQLTools 插件。

## 🧩 数据建模与 ETL 逻辑 (Data Modeling & Star Schema)

在本项目中，我们通过 `data_factory.py` 将原始的“大宽表”转化成了工业标准的**星型模型 (Star Schema)**。这是数据分析师迈向数据工程师的关键一步。

### 1. 拆分标准：事实 (Fact) 与 维度 (Dimension)
*   **维度表 (`company_dim`, `skills_dim`)**：
    *   **逻辑**：存储“属性”。将重复出现的文字（如公司名、技能名）提取出来，存为唯一的清单。
    *   **标准**：每一行代表一个唯一的实体，拥有一个唯一的 **ID (Primary Key)**。
*   **事实表 (`job_postings_fact`)**：
    *   **逻辑**：存储“度量”。记录职位、薪资、日期等核心分析对象。
    *   **标准**：不再存储冗余的文字，而是存储指向维度表的 **ID (Foreign Key)**，从而极大地压缩空间并提升查询效率。

### 2. Pandas 实现 ETL 的“三板斧”
脚本 `data_factory.py` 严格遵循以下工业标准流程：
1.  **提取与去重 (Deduplication)**：利用 `drop_duplicates()` 从 78 万行数据中淘洗出唯一的公司和技能清单。
2.  **ID 赋能 (ID Generation)**：利用 `range` 或 `index` 为每个唯一实体生成数字 ID，作为数据库索引的基石。
3.  **映射回填 (Mapping)**：利用 `merge()` (类似 SQL JOIN) 将生成的 ID 关联回原始数据，并剔除掉冗余的文字列（如 `company_name`）。

### 3. 为什么这样做？
*   **空间优化**：将 200MB+ 的文本数据通过数字 ID 化，显著降低了存储压力。
*   **性能飞跃**：数据库检索数字 ID 的速度远快于匹配长字符串。
*   **逻辑清晰**：符合 Luke 课程要求的分析结构，使得复杂的 JOIN 查询成为可能。

## 📂 项目模块说明

*   **`data/`**: 存放原始 CSV 数据（数据存档）。
*   **`data_factory.py`**: 数据工厂。负责将 220MB 的宽表拆解成符合 SQL 范式的 4 张标准表。
*   **`scripts/`**: 分析实验室。包含 5 个核心分析 SQL，解决从“高薪职位”到“性价比最高技能”的业务问题。

## 🚦 快速开始

1. **环境准备**：启动 Docker 容器 `post_study`。
2. **数据同步**：运行 `data_factory.py` 将 78 万行数据灌入 `sql_course` 数据库。
3. **拨号连接**：在 SQLTools 里新建连接，指向 `localhost:5432`，数据库填 `sql_course`。
4. **开始分析**：依次运行 `scripts/` 目录下的分析脚本。

---
